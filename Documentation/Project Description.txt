ToDo лист

Разработать простейший ToDo лист.

Имеется сущность - Легенда/Legend.
Она содержит поля:

ID - integer
Идентификатор записи.

Header - varchar(64)
Заголовок записи.

Description - varchar(256)
Описание записи.

Is_Done - bool default false
Состояние выполнения записи. Булевое.
По умолчанию - не выполнено.

Status - char(1)
Статус записи. Одиночный символ.
Отображает последнее действие над записью.
A - добавлена
U - изменена
D - удалена

Create_Date - date
Дата создания записи. 
Вызывается функция из базы данных,
заполняющая текущее время.

Update_Date - date
Дата изменения записи.
Вызывается функция из базы данных,
заполняющая текущее время.
При добавлении записи совпадает с Create_Date.

Первая инициализация таблицы записей:

Legend
-----------------------------------------------------------------------------------------------
	ID		Header		Description		Is_Done		Status		Create_Date		Update_Date		


-----------------------------------------------------------------------------------------------
ToDo :

Расширяется до заметок, напоминаний, календарь, распорядок дня
Задать патерны поведения с гибкой настройкой

Нужен порядок записей. Возможно, для будущей иерархии записей

Нужны операции :
добавить запись
изменить запись
удалить запись
получить запись
получить список записей
